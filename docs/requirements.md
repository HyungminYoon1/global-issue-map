# Global Issue Map 

## 요구사항 정의서

---

## 1. 프로젝트 개요

전쟁, 경제, 자연재해, 정치 등 세계 주요 이슈를 지도 기반으로 시각화하여 사용자가 세계 상황을 한눈에 파악할 수 있도록 하는 웹 서비스이다.
또한 사용자가 관심 있는 기사를 저장해 다시 볼 수 있는 **나만의 기사 기능**을 제공한다.
**한국어/영어 2개 국어**를 지원하여 국내외 사용자 모두 이용 가능하다.

---

## 2. 서비스 구조

서비스는 다음 6개 페이지로 구성된다.

* 홈
* 전쟁
* 경제
* 자연재해
* 정치
* 나만의 기사

---

## 3. 홈 기능

홈은 세계 이슈를 종합적으로 보여주는 메인 대시보드 역할을 한다.

### 3.1 세계 이슈 지도

* 세계 지도 위에 주요 이슈를 핀 형태로 표시한다.
* 핀 색상을 통해 카테고리를 구분한다.
* 대륙별 필터를 통해 특정 대륙의 이슈만 조회할 수 있어야 한다.

핀 색상 구분

* 전쟁: 빨강 (#EF4444)
* 경제: 초록 (#16A34A)
* 정치: 노랑 (#EAB308)
* 자연재해: 주황 (#F97316)
* 기타: 회색 (#94a3b8)

* 사용자는 핀 클릭 시 뉴스 요약 정보를 확인할 수 있다.
* 동일 좌표에 여러 핀이 있을 경우 겹침을 방지하여 원형으로 분산 배치한다.
* 핀 위에 마우스를 올리면 크기가 확대되고 강조 효과가 적용된다.

### 3.2 뉴스 검색 기능

* 사용자는 키워드를 입력하여 원하는 뉴스를 검색할 수 있어야 한다.
* 검색 결과는 리스트와 지도 핀으로 함께 확인할 수 있어야 한다.
* 대륙, 카테고리 조건을 조합하여 검색 결과를 필터링할 수 있어야 한다.

### 3.3 주요 뉴스 카드

* 현재 가장 영향력 있는 뉴스 5개를 카드 형태로 제공한다.
* 카드에는 제목, 지역, 카테고리, 언론사, 요약 정보가 포함되어야 한다.

### 3.4 언어 전환

* 헤더 네비바에 KR/EN 토글 버튼을 제공한다.
* 사용자는 한국어/영어 간 전체 UI를 전환할 수 있어야 한다.
* 네비게이션, 검색, 필터, 버튼 등 모든 UI 라벨이 선택 언어로 표시된다.
* 기사 제목/요약은 한국어(GPT 번역) 또는 영어(GDELT 원문)로 전환된다.
* 언어 설정은 브라우저 localStorage에 저장되어 재방문 시에도 유지된다.

## 4. 카테고리 페이지 기능

전쟁, 경제, 자연재해, 정치 페이지는 해당 주제의 뉴스만 집중적으로 보여준다.

### 4.1 지도 기반 뉴스 표시

* 해당 카테고리의 뉴스만 지도에 핀으로 표시한다.
* 뉴스 중요도에 따라 핀의 크기, 색상 강조, 시각 효과를 다르게 표현한다.
* 이를 통해 사용자는 중요한 뉴스를 직관적으로 구분할 수 있어야 한다.
* 대륙별 필터를 통해 특정 대륙의 뉴스만 조회할 수 있어야 한다.

예시

* 중요도 높은 뉴스: 큰 핀, 진한 색상
* 중요도 보통 뉴스: 중간 크기 핀
* 중요도 낮은 뉴스: 작은 핀, 옅은 색상

### 4.2 뉴스 상세 정보

* 핀 클릭 시 뉴스 상세 정보를 확인할 수 있어야 한다.

포함 정보

* 뉴스 제목
* 발생 지역
* 언론사
* 발행일
* 뉴스 요약

### 4.3 AI 분석 정보

뉴스 선택 시 다음 정보를 제공한다.

* AI 해석
* 예상 동향
* 미치는 영향

영향 예시

* 전쟁: 금, 석유, 주식
* 경제: 환율, 금리, 시장
* 자연재해: 공급망, 교통, 산업
* 정치: 외교, 무역, 시장 심리

### 4.4 기사 저장 기능

* 사용자는 카테고리 페이지에서 원하는 뉴스를 저장할 수 있어야 한다.
* 저장 버튼을 통해 관심 기사를 나만의 기사 페이지로 보낼 수 있어야 한다.

## 5. 나만의 기사 페이지 기능

나만의 기사 페이지는 사용자가 저장한 뉴스만 모아보는 개인화 페이지이다.

### 주요 기능

* 저장한 기사 목록 확인
* 기사 제목, 카테고리, 대륙, 지역, 언론사, 요약 정보 표시
* 저장한 기사 상세 보기
* 저장 기사 삭제 기능
* 카테고리별, 대륙별로 저장 기사 분류 가능

### 목적

* 사용자가 관심 있는 뉴스만 따로 모아 다시 볼 수 있도록 한다.
* 단순 조회형 서비스가 아니라 개인 맞춤형 탐색 경험을 제공한다.

## 6. 데이터 요구사항

뉴스 데이터는 외부 뉴스 API(GDELT, NewsData.io, GNews)를 통해 수집한다.
GDELT에서는 **키워드 검색 + GKG 주제(Theme) 기반 검색**을 합집합(OR)으로 결합하여 검색 누락을 최소화한다.
수집된 기사는 **원문 URL에서 리드 문단(300자)을 추출**한 뒤, GPT에 제목+본문을 함께 전달하여 한국어 번역 및 5개 카테고리(war, economy, disaster, politics, others) 재분류를 수행한다.

### 데이터 수집 정책

* 수집 주기: 3시간 (하루 8회)
* GDELT 검색 전략: 확장 키워드 + GDELT Theme 코드 합집합 쿼리
* 표시 범위: 최근 48시간 이내 수집된 기사만 조회 대상
* 자동 만료: 30일 초과 기사는 TTL 인덱스로 자동 삭제
* others 카테고리: 홈/대륙별/검색 조회에서 자동 제외 (카테고리 전용 페이지에서만 접근 가능)

### 카테고리 분류 기준

* **war (전쟁)**: 국가 간 전쟁, 내전, 군사 충돌, 조직적 테러, 군사 작전 (개인 총격·강도 등 일반 범죄 제외)
* **economy (경제)**: 경제, 무역, 금융, 주식, 환율, 물가, 관세, 경기침체
* **disaster (자연재해)**: 자연재해, 기후, 환경 오염, 전염병, 대형 사고
* **politics (정치)**: 정치, 외교, 선거, 정상회담, 제재, 법안, 정부 정책
* **others (기타)**: 위 4개에 해당하지 않는 기사 (범죄, 사건사고, 연예, 스포츠, 라이프스타일 등)

### 뉴스 데이터

* 뉴스 제목 (한국어 번역 + 원문 보존)
* 언론사
* 발행일
* 뉴스 요약 (한국어 번역 + 원문 보존)
* 국가 또는 지역
* 대륙
* 카테고리 (GPT 재분류 결과)
* 키워드
* 지도 좌표
* 뉴스 중요도 정보
* 기사 원문 URL

### AI 분석 데이터

* 뉴스 해석
* 예상 동향
* 영향 분석

### 사용자 저장 데이터

* 저장한 기사 ID
* 기사 제목
* 카테고리
* 대륙
* 지역
* 언론사
* 요약
* 저장 시간

## 7. MVP 기능

최소 구현 기능은 다음과 같다.

* 홈 대시보드
* 지도 핀 표시
* 핀 색상 카테고리 구분
* 대륙별 뉴스 필터
* 뉴스 검색 기능 (키워드, 대륙, 카테고리 필터)
* 카테고리별 뉴스 페이지
* 중요도에 따른 핀 크기/강조 차등 표현
* 기사 저장 기능
* 나만의 기사 페이지
* AI 분석 정보 제공
